# FE Specs

Frontend specifications created by `/write-fe-spec`. Each spec corresponds to one or more IA shards and defines the complete frontend contract — components, state management, accessibility, responsive behavior, and performance budgets.

## Naming Convention

Specs are numbered to match their source IA shard: `00-*.md`, `01-*.md`, ..., `NN-*.md`. An `index.md` file maps spec numbers to IA shard sources.

## Completion Requirement

All FE specs must reach **Complete** status before `/plan-phase` begins. Run `/audit-ambiguity fe` to verify.
